Django Interactive

Headstart your modern and dynamic web application.

Django Interactive is library that aims to make the connection between client side and server side easier by letting the developer define view hierarchies and auto-generating client-side methods that can convenient retrieve and rerender parts of the page.

Getting Started

Install the module from the Python registry.
  pip install django-interactive
In settings.py, add django-interactive to your installed apps.
  INSTALLED_APPS = (
    ...
    'django-interactive',
    ...
  )

How to use?

In a traditional django action method, the application is clueless about what happens after calling render and returning the Http response. django-interactive let's you create a view hieararch by defining page views and sub-views that can be nested with each other. This way the application can know the computations pertaining certain parts of a page.

1. Define Your Views

There are two types of views: Page Views and Subviews.


def render_list(request, context = {}):
    # unique id of wrapper element
    context['id'] = "unique-list"
    # el defaults to 'div'
    context['el'] = "ol"
    # optional value
    context['classname'] = "my-list"
    # any other value
    context['items']  = Item.objects.all()

    return render_subview(request, 'list.html', context)

layout_reg.register_subview(render_list)

To define subview, the method needs to return render_subview(request, TEMPLATE_NAME, context) and should also be registered by calling layout_reg.register_subview(METHOD_NAME) 

def home(request):

    context = {}
    return render_pageview(request, 'index.html', context)

To define a Page View, you only need to return render_pageview(request, TEMPLATE_NAME, context)

2. Use Generated Javascript Functions

Once you define the subviews in your Django methods, the autogenerated functions are automagically added to your page. There are currently three generated methods for every subview.

// This will seamlessly reload the 
// list on the page.
Stub.render_list_reload();
METHOD_NAME_reload
Rerenders the up-to-date html dynamically and replaced the view on the page. 
// Will print the up-to-date html of
// the list subview
Stub.render_list_html(function(html_string) {
  console.log(html_string);
});

/*
Expected output:
<ul>
  <li>Item 1</li>
  <li>Item 2</li>
  <li>...</li>
</ul>
*/
METHOD_NAME_html
Returns the up-to-date html as a parameter to the callback function. (asynchronous function) 
// Will print data pertaining the list
// in JSON form.
Stub.render_list_json(function(data) {
  console.log(data);
});

/*
Expected output:
[
  {
    "title": "Item 1",
    "status": "Incomplete"
  },
  {
    "title": "Item 2",
    "status": "Complete"
  },
  {
    ...
  }
]
*/
METHOD_NAME_json
Returns the up-to-date json from the view as a parameter to the callback function. (asynchronous function)
